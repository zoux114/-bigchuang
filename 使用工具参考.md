# RAG 系统安装工具快速参考

本项目包含多个自动化工具，帮助你快速安装和配置 RAG 系统。

## 🎯 工具概览

| 工具 | 功能 | 使用场景 |
|------|------|--------|
| **install.py** / **install.bat** | 一键安装所有依赖 | 首次安装/重新配置 |
| **doctor.py** | 系统诊断和环保检查 | 遇到问题时诊断 |
| **quickstart.py** / **run.bat** | 交互式启动菜单 | 日常开发/部署 |

---

## 📋 按步骤使用

### 第一步：运行一键安装

选择适合你的操作系统：

#### **Windows - 方式 A（推荐）**
双击运行：
```
install.bat
```

#### **Windows - 方式 B**
在 PowerShell 中运行：
```powershell
python install.py
```

#### **Windows - 启动菜单（可选）**
```powershell
python quickstart.py
# 或
run.bat
```

#### **macOS / Linux**
```bash
python install.py
```

### 第二步：检查安装结果

如果安装过程有警告，运行诊断：

```bash
python doctor.py
```

### 第三步：配置 API 密钥

编辑 `config/.env` 文件：

```bash
# Windows
notepad config\.env

# macOS/Linux
nano config/.env
```

找到以下行并填入你的 API 密钥：

```env
LLM_API_KEY=你的_API_密钥
LLM_BASE_URL=https://spark-api-open.xf-yun.com/v2/
LLM_MODEL=spark-x
```

### 第四步：准备数据

将 PDF/DOCX 文件放入 `data/raw/` 目录

### 第五步：初始化向量数据库

```bash
python ingest.py
```

### 第六步：启动服务

#### 方式 A：使用菜单（推荐）
```bash
python quickstart.py   # Python 版（跨平台）
# 或
run.bat               # Windows 版
```

然后选择选项 6

#### 方式 B：手动启动

在两个终端窗口中运行：

```bash
# 终端 1: API 服务器
python api_server.py
# 访问: http://localhost:8001

# 终端 2: 前端
cd frontend
npm run dev
# 访问: http://localhost:5173
```

## 🔧 每个工具详解

### install.py / install.bat

**功能**：一键安装和配置

**执行内容**：
- ✓ 检查 Python 版本 (需要 3.8+)
- ✓ 检测 GPU 支持
- ✓ 创建目录结构
- ✓ 创建 `.env` 配置文件
- ✓ 安装 Python 依赖
- ✓ 安装 Node.js 依赖
- ✓ 构建前端
- ✓ 验证安装

**输出示例**：
```
✓ Python 版本: 3.11.0
✓ GPU 可用: NVIDIA GeForce RTX 3090 (CUDA 12.1)
✓ 目录就绪: data/raw
✓ .env 文件已创建
✓ Python 依赖安装完成
✓ 前端依赖安装完成
✓ 前端构建成功
```

**常见错误处理**：
- 如果 GPU 检查失败 → 检查 CUDA 安装
- 如果前端依赖失败 → 检查 Node.js 安装
- 如果依赖冲突 → 运行 `pip install --upgrade pip`

**使用场景**：

### quickstart.py / run.bat

**功能**：交互式启动菜单

**菜单选项**：
```
1. 一键安装 (首次使用必选)
2. 诊断系统
3. 初始化向量数据库
4. 启动 API 服务器
5. 启动前端开发服务器
6. 启动完整应用 (API + Web)
7. 打开项目文件夹
8. 编辑环境配置 (.env)
9. 查看帮助文档

```

**推荐工作流**：
```
1. 首次安装 → 选择 1
2. 遇到问题 → 选择 2
3. 日常开发 → 选择 6
4. 修改配置 → 选择 8
```

## 🆘 故障排除

### 问题 1：Python 版本不符

```bash
Python 版本过低: 3.7.0，需要 3.8+
```

**解决方案**：
- 从 [python.org](https://www.python.org/) 下载 Python 3.11
- 重新运行 install.py

### 问题 2：模块冲突

```bash
ERROR: pip's dependency resolver does not currently take into account
```

**解决方案**：
```bash
pip install --upgrade pip setuptools
python install.py
```

### 问题 3：GPU 未被识别

```bash
⚠ GPU 不可用，将使用 CPU 进行 Embedding
```

**确保**：
- 安装了 NVIDIA 显卡驱动
- 安装了 CUDA Toolkit 12.1+
- 安装了 cuDNN

**手动配置**：
```bash
# 卸载现有 PyTorch
pip uninstall torch torchvision torchaudio

# 安装 GPU 版本
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
```

### 问题 4：前端无法访问

```bash
⚠ Node.js 未安装
```

**解决方案**：
- 从 [nodejs.org](https://nodejs.org/) 下载 Node.js
- 重新运行 install.py

### 问题 5：磁盘空间不足

```bash
✗ 硬盘空间 5.00 GB 可用（不足）
```

**解决方案**：
- 清理磁盘，至少保留 20 GB 空间
- 模型会缓存在 models/ 目录

### 问题 6：API 密钥配置错误

如果看到认证错误：
```bash
python doctor.py        # 检查 .env 是否已读取
nano config/.env        # 编辑文件
```
